Texto de apoio


S -> Single Responsibility Principle: Classes possuem apenas uma função.

O -> Open-Closed Principle: Classes são expansíveis, não modificáveis.

L -> Liskov Substitution Principle: Um instância pode ser substituída por um subtipo.

I -> Interface Segregation Principle: Interfaces específicas são melhores que gerais.

D -> Dependency Inversion Principle: Dependências só podem ocorrer com abstrações.
